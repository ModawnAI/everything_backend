

주요 기능 명세
1. 로그인 화면
주요 접근 권한: 플랫폼 관리자, 샵 관리자
화면 구성: 이메일/비밀번호 로그인 폼, 비밀번호 찾기 링크, "샵 입점 신청하기" 버튼(구글 폼 연동).
연결 화면: 로그인한 관리자 역할에 따라 맞춤형 대시보드로 이동.
2. 대시보드
주요 접근 권한: 플랫폼 관리자, 샵 관리자
역할별 메뉴 구성:
[플랫폼 관리자]: 전체 현황, 회원 관리, 뷰티샵 관리, 예약 관리(모니터링), 콘텐츠 관리, 포인트 관리, 공지사항 관리, 푸시 발송 관리, 통계/리포트, 시스템 설정
[샵 관리자]: 내 샵 대시보드, 내 샵 관리, 서비스 관리, 예약 관리, 차액 결제 요청 관리, 고객 관리, 정산 관리
3. 회원 관리 화면
주요 접근 권한: 플랫폼 관리자
화면 구성: 회원 검색(이메일, 닉네임) 및 필터 기능. 회원 목록 테이블(ID, 닉네임, 가입일, 보유 포인트, 인플루언서 여부 등).
관리 기능: 상세 보기, 계정 정지/활성화, 포인트 수동 조정, 인플루언서 지정/해제. 상세 보기 내에서 추천 친구 목록 및 각 친구의 '첫 결제 여부' 확인 기능.
기본 기능: 인플루언서 자격 조건('친구 50명 초대 및 전원 첫 결제 완료') 충족 여부를 확인하고 관리.
4. 뷰티샵 관리 화면
주요 접근 권한: 플랫폼 관리자, 샵 관리자
화면 구성 (역할별):
[플랫폼 관리자]: 전체 샵 목록 조회, 입점 신청 목록 조회 및 심사(승인/거절) 처리.
[샵 관리자]: 본인 샵의 모든 정보(소개글, 사진, 정산 계좌 등) 직접 수정 및 관리. '가게 메시지 보내기' 링크(예: 카카오톡 채널 URL) 입력 필드 제공.
5. 예약 관리 화면
역할별 기능 차이:
[플랫폼 관리자]: 플랫폼 전체 예약 현황을 세분화된 상태값에 따라 실시간 모니터링만 가능 (실무 처리 불가).
[샵 관리자]: 본인 샵의 모든 예약을 직접 관리.
'예약 요청'에 대해 [확정] 또는 [거절] 처리.
시술 완료 후, 해당 예약 건에 대해 [방문 완료] 처리.
'방문 완료'된 건에 대해 사용자에게 [차액 결제 요청] 버튼 클릭.
예약 상태 정의: 예약 요청 → 예약 확정 → 시술 완료 → 최종 완료 (사용자의 차액 결제 완료 시)
기본 기능: 샵 관리자가 예약 상태를 변경하면 해당 사용자에게 자동으로 푸시 알림이 발송됩니다.
6. 포인트 정책 및 로그 관리
주요 접근 권한: 플랫폼 관리자
화면 구성 (정책 설정):
예약 적립률 (%)
추천인 적립률 (일반/인플루언서 차등 설정)
포인트 적립 한도 (결제액 기준, 예: 300,000원)
포인트 사용 가능 시점 (적립 후 N일, 예: 7일)
포인트 조정 감사 로그(Audit Log)
기본 기능: 플랫폼의 모든 포인트 정책을 수립하고, 예외적인 포인트 지급/차감 처리 및 모든 내역을 추적.

8. 고객 관리 화면
주요 접근 권한: 샵 관리자
기본 기능: 본인 샵 방문 고객의 방문 이력, 선호 서비스 조회, 응대 메모 작성. 고객 상세 정보에서 예약금, 차액 결제 내역 등 전체 결제 히스토리 조회 가능.
9. 통계/리포트 화면
주요 접근 권한: 플랫폼 관리자
화면 구성:
결제 통계: 기간별 전체 매출, 예약금 합계, 차액 결제 합계.
콘텐츠 통계: 일/주/월별 게시물 등록 수, 사용자 참여(좋아요) 수.
회원 통계: 전체 회원 수, 신규 가입자 수, 인플루언서 수.
예약 및 매출 통계: 샵별 예약 및 매출 순위.
10. 시스템 설정 화면
주요 접근 권한: 플랫폼 관리자
화면 구성: 앱 설정(버전 관리, 점검 모드), 결제 설정(토스페이먼츠 API 연동 정보), 외부 연동 API 키 관리 등.

화면 연결 구조
로그인 → 대시보드 (역할별)
├── 회원 관리 (플랫폼 관리자)
├── 샵 관리
│ ├── 입점 신청 관리 (플랫폼 관리자)
│ └── 샵 정보 관리 (플랫폼 관리자, 샵 관리자)
├── 예약 관리 (모니터링: 플랫폼 관리자 / 실무: 샵 관리자)
├── 콘텐츠 관리 (플랫폼 관리자)
├── 포인트 관리 (플랫폼 관리자)
├── 푸시 발송 (플랫폼 관리자)
├── 고객 관리 (샵 관리자)
├── 통계/리포트 (플랫폼 관리자)
└── 시스템 설정 (플랫폼 관리자)
