# Archive Directory

**Created:** 2025-11-10

This directory contains archived documentation, test files, SQL scripts, and data files that were previously in the root directory of the project.

## Directory Structure

### `/docs` - Documentation Files
Contains all historical documentation, phase completion reports, test results, API guides, and implementation summaries. These files document the development history of the project.

### `/tests` - Test Scripts
Contains all test shell scripts (.sh) and test JavaScript files (.js) that were used during development and testing phases.

### `/sql` - SQL Files
Contains SQL migration scripts, manual database operations, and SQL-based utilities.

### `/data` - JSON Data Files
Contains JSON data files including shop data, geocoding results, test credentials, and OpenAPI specifications.

## Preserved Files (NOT Archived)

The following files remain in the root directory as they are essential for project operation:

- `CLAUDE.md` - Project instructions for Claude Code
- `package.json`, `package-lock.json` - Node.js dependencies
- `tsconfig.json` - TypeScript configuration
- `.mcp.json` - MCP server configuration
- `nodemon.json` - Nodemon configuration
- `google-services.json` - Firebase configuration
- `opencode.json` - OpenCode configuration

## Purpose

This archive was created to:
1. Clean up the root directory for better project organization
2. Preserve historical documentation and test files
3. Maintain access to legacy scripts and data
4. Improve development workspace clarity

## Accessing Archived Files

All files are preserved and can be accessed when needed:
- Reference historical docs: `archive/docs/`
- Run old test scripts: `archive/tests/`
- Review SQL migrations: `archive/sql/`
- Access data files: `archive/data/`
